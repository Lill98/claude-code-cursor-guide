# Example: TDD Workflow — Publish Post (blog-api)

End-to-end walkthrough of the TDD workflow using a real feature from `blog-api`. Shows how to go from a spec file to skipped tests to passing tests, with automatic test feedback after each Claude task.

**Stack:** FastAPI + SQLAlchemy + PostgreSQL + Pydantic v2 + pytest + Alembic

---

## The Spec

The spec below would normally be generated by `/research-ticket BLOG-14`. It is shown inline here so you can see exactly what the workflow starts from.

```markdown
# Spec: BLOG-14 — Publish Post

## Overview
Authors can promote their draft posts to published state.
Once published, a post becomes publicly visible and cannot be re-published.

## Acceptance Criteria
- [ ] AC01: Only the post author (or admin) can publish a post
- [ ] AC02: Only posts with status "draft" can be published — re-publishing is rejected
- [ ] AC03: Publishing sets status to "published" and records a published_at timestamp
- [ ] AC04: Returns 404 if the post does not exist or has been soft-deleted
- [ ] AC05: Returns 403 if the requesting user is not the author or an admin
```

## Step 5: Implement the Method

Ask Claude to implement `publish_post` one behavior at a time:

```
Implement publish_post in app/services/post_service.py to make the skipped tests pass.
Query the post with deleted_at.is_(None), check ownership, enforce draft-only rule,
then set status="published" and published_at=datetime.utcnow().
```

**Completed `app/services/post_service.py`:**

```python
from datetime import datetime

from fastapi import HTTPException
from sqlalchemy.orm import Session

from app.models import Post, User


class PostService:
    def publish_post(self, db: Session, post_id: int, current_user: User) -> Post:
        post = db.query(Post).filter(Post.id == post_id, Post.deleted_at.is_(None)).first()
        if not post:
            raise HTTPException(status_code=404, detail="Post not found")
        if post.author_id != current_user.id and current_user.role != "admin":
            raise HTTPException(status_code=403, detail="Not authorized")
        if post.status == "published":
            raise HTTPException(status_code=409, detail="Post is already published")
        post.status = "published"
        post.published_at = datetime.utcnow()
        db.commit()
        db.refresh(post)
        return post
```

## Step 6: Implement Tests One by One

Remove `@pytest.mark.skip` from each test and fill in the assertions. Three completed examples:

```python
import pytest
from unittest.mock import MagicMock, patch
from datetime import datetime
from fastapi import HTTPException
from sqlalchemy.orm import Session

from app.models import Post, User
from app.services.post_service import PostService


@pytest.fixture
def service():
    return PostService()


@pytest.fixture
def mock_db():
    return MagicMock(spec=Session)


@pytest.fixture
def mock_author():
    user = MagicMock(spec=User)
    user.id = 1
    user.role = "author"
    return user


@pytest.fixture
def mock_admin():
    user = MagicMock(spec=User)
    user.id = 2
    user.role = "admin"
    return user


@pytest.fixture
def mock_reader():
    user = MagicMock(spec=User)
    user.id = 3
    user.role = "reader"
    return user


@pytest.fixture
def mock_post():
    post = MagicMock(spec=Post)
    post.id = 10
    post.author_id = 1
    post.status = "draft"
    post.deleted_at = None
    return post


class TestPublishPost:
    def test_author_can_publish_own_draft(self, service, mock_db, mock_author, mock_post):
        mock_db.query.return_value.filter.return_value.first.return_value = mock_post

        result = service.publish_post(mock_db, post_id=10, current_user=mock_author)

        assert mock_post.status == "published"
        assert isinstance(mock_post.published_at, datetime)
        mock_db.commit.assert_called_once()
        mock_db.refresh.assert_called_once_with(mock_post)
        assert result is mock_post

    def test_admin_can_publish_any_post(self, service, mock_db, mock_admin, mock_post):
        # post.author_id=1, admin.id=2 — different user, but admin role bypasses ownership
        mock_db.query.return_value.filter.return_value.first.return_value = mock_post

        result = service.publish_post(mock_db, post_id=10, current_user=mock_admin)

        assert mock_post.status == "published"
        assert result is mock_post

    def test_raises_403_for_reader(self, service, mock_db, mock_reader, mock_post):
        # post.author_id=1, reader.id=3 — neither owner nor admin
        mock_db.query.return_value.filter.return_value.first.return_value = mock_post

        with pytest.raises(HTTPException) as exc_info:
            service.publish_post(mock_db, post_id=10, current_user=mock_reader)

        assert exc_info.value.status_code == 403
```

## Step 9: Pre-commit Hook with pre-commit Library

Install `pre-commit` to enforce the test gate at commit time — this applies to all team members, not just Claude Code users:

```bash
pip install pre-commit
```

**`.pre-commit-config.yaml`** (add to repo root):

```yaml
repos:
  - repo: local
    hooks:
      - id: pytest
        name: pytest
        entry: pytest --tb=short -q
        language: system
        pass_filenames: false
```

Enable the hook:

```bash
pre-commit install
git add .pre-commit-config.yaml && git commit -m "chore: add pre-commit pytest gate"
```

**When all tests pass**, `git commit` works normally:

```
$ git commit -m "feat(post): implement publish_post"
[pytest] ........................................
pytest...................................................................Passed
[main a3f9c12] feat(post): implement publish_post
```

**When a test fails**, the commit is rejected:

```
$ git commit -m "feat(post): implement publish_post"
[pytest] ........................................
FAILED tests/test_post_service.py::TestPublishPost::test_raises_conflict_when_already_published
- AssertionError: assert 409 == 409 ... HTTPException not raised

pytest...................................................................Failed
- hook id: pytest
- exit code: 1
```

Fix the failing test, re-run `git commit`. The gate ensures broken code never lands on the main branch.
